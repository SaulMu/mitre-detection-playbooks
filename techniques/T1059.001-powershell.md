# T1059.001 — PowerShell

## Overview

PowerShell is commonly used by administrators, but attackers also abuse it for execution, payload download, persistence and defense evasion.

## Tactic

- Execution

## Data Sources

- Windows Event Logs
- Sysmon Event ID 1: Process Creation
- PowerShell logs
- EDR telemetry
- SIEM events

## Suspicious Patterns

- Use of `-EncodedCommand`
- Use of `-ExecutionPolicy Bypass`
- Download commands such as `Invoke-WebRequest` or `IEX`
- PowerShell launched by Office applications
- PowerShell spawning from unusual parent processes

## Detection Logic

Look for PowerShell execution with suspicious command-line arguments.

Example logic:

```text
process_name = powershell.exe
AND command_line contains "-EncodedCommand"
OR command_line contains "-ExecutionPolicy Bypass"
OR command_line contains "IEX"
```

## Investigation Steps
1. Identify the user and host.
2. Review the full command line.
3. Decode encoded commands.
4. Analyze parent and child processes.
5. Check network connections.
6. Search for similar activity across the environment.
7. Determine if the behavior is administrative or malicious.

## False Positives
- System administration scripts
- Software deployment tools
- Monitoring agents
- IT automation tasks
- 
## Response Recommendations
- Isolate host if malicious execution is confirmed.
- Disable compromised user account if needed.
- Block malicious domains or IPs.
- Improve detection logic based on confirmed evidence.
